Design Principles — Airbnb-Inspired Onboarding

Key patterns extracted from Airbnb's approach, adapted for Rosie's context.

1 Progressive Disclosure
One question per screen. Name → Birthday → Done. Never overwhelm a sleep-deprived parent with a wall of form fields.
2 Emotional-First Copy
Lead with feeling, not features. "Your calm companion" > "Baby tracker app". Parents are buying peace of mind.
3 Minimal Required Data
Only ask what's needed to start: baby name + birthday. Gender, weight, preferences can be added later in Profile. Reduce drop-off.
4 Personalize Immediately
"When was Rosie born?" — use the baby's name as soon as it's captured. Builds emotional connection with each step.
5 Progress = Motivation
4-segment progress bar shows the full journey upfront. Purple gradient for current, green for done. Users see the finish line.
6 Celebrate Completion
🎉 success screen with green checkmarks. Gamification creates a micro-dopamine hit. User's first impression of the app is accomplishment.
