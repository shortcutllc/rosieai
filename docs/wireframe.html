1 Floating Chat Pill
Chat moves from a tab to a floating pill at the bottom. Always visible, always accessible regardless of which tab is active. Thumb-friendly for one-handed use.
2 3-Tab Layout
Removing Ask Rosie from the tab bar gives each tab more room. Home, Timeline, This Week — all content tabs. Chat is a universal overlay, not a content tab.
3 Airbnb Search Bar Pattern
Like Airbnb's search pill, the chat prompt is the primary CTA. It floats over content without displacing it. Tapping opens fullscreen chat overlay (same animation).
4 Contextual Prompt
The pill text can be contextual: "Ask about Week 14...", "Ask about Rosie's sleep...", adapting to the current tab and data context.

1 Consistent Time Inputs
Custom styled time fields replace native <input type="time">. Same border-radius (12px), focus state (blue border + translateY(-1px) lift), and typography across all modals.
2 Bottom Sheet Pattern
Modal slides up from bottom as a sheet. Drag handle at top, backdrop blur. Consistent with iOS patterns and our animation thesis (400ms decelerate enter).
3 Duration Display
Auto-calculated duration shown in a colored card between time inputs and note. Type-specific color coding (orange=feed, purple=sleep, green=diaper).
4 Gradient Save Button
Full-width save button with type-specific gradient. Consistent 14px border-radius, 17px/700 text. Active state: scale(0.98) per thesis interactive feedback spec.

1 Full-Width Header Bar
Header spans the entire viewport width with 32px side padding (Airbnb pattern). Weather far-left, profile avatar far-right. Creates visual breadth on wide screens.
2 Centered Content Column
Content below header stays at 600px max-width, centered. This gives the reading area focus while the header provides global navigation context.

Floating pill shows keyboard shortcut (⌘K) on desktop. Same position as mobile but with wider max-width (400px) and shortcut hint.
4 Subtle Header Border
On desktop, a faint 1px bottom border separates the full-width header from centered content. On mobile, the compact 48px height makes this unnecessary.

Design Thesis Reference

Key tokens and patterns applied across all wireframes above.

Spacing Grid
xs: 4px
sm: 8px
md: 12px (card gap)
lg: 16px (mobile padding)
xl: 20px (card padding)
2xl: 24px (section gap)
3xl: 32px (desktop padding)
Typography
Hero: 28px/700
Section title: 18px/700
Card title: 14-16px/600
Body: 14-15px/400-500
Secondary: 13px/400
Labels: 11px/700 UPPER
Stats: 18px/700 colored
Animation Curves
Enter: 400ms decelerate
Exit: 300ms accelerate
Standard: 200ms ease
Spring: 300ms overshoot
Stagger: 60ms delay each
Reduced motion: none
