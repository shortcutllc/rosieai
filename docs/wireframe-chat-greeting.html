1 Living Heart
Bounces in on entrance, then settles into a gentle breathing loop — subtle scale (1.0→1.04), 2px float, and soft purple glow. 4-second cycle. Feels alive, like a calm presence. Not a loading spinner.
2 Mom-First Greeting
Typewriter types "Hey Lauren — how are you?" Focused on the parent, not the baby. The friend who asks about YOU first. Copy rotates daily: "Morning. You made it." / "Still up? I'm right here." / "You did a lot today."
3 Validating Subtext
Subtext centers the parent's experience: "You're doing amazing. Seriously." Not baby milestones, not app features. Validation first, then an invitation to talk. Changes by age/weather/time.
4 Quick Actions (Unchanged)
Three one-tap shortcuts stay. Quick log, How's baby?, Activity ideas. Functional, always visible, compact. These are actions, not suggestions.
A Compact Pills
Suggestions as small wrapping pills. No icons, no arrows, no bold. Visually quiet — greeting stays the hero. Short labels that feel like topics, not full questions. 13px, muted color.
B Ghost Text
Suggestions as plain text in quotes. Lowest visual weight possible — like whispered conversation starters. No cards, no borders. Feels like reading, not choosing. "Or try..." instead of "TRY ASKING."

Quick Log — Separate & Prominent

Logging feeds, diapers, and naps is the #1 user action (10-15x/day). Each gets its own button with instant color recognition. "How's Rosie?" and "Activity ideas" move to the suggestion area — they're chat prompts, not log actions.

C Color Action Cards
Mini cards with category color accents. Reuses the home screen visual language — parents already know these cards. Explicit "+ Log feed" buttons feel tappable and directional.
D Icon Circles
iOS-style quick actions. Big 56px tap targets with tinted backgrounds. "Time since last" adds context without pressure — you see "2h ago" and think "oh, time for a feed."
E Stacked List
Most informational. Shows last event details inline (what, when, how much). Best for parents who want context before tapping. Chevrons signal "tap to open." Takes more vertical space.
1 Separate Log Actions
Each event type (feed, sleep, diaper) gets its own button with its category color. No more guessing what "Quick log" does. One tap opens the right modal.
2 Category Colors
Orange = feed, purple = sleep, green = diaper. These colors are used everywhere in the app — timeline, stats, quick log modals. Instant recognition, zero reading required.
3 Chat Prompts Demoted
"How's Rosie?" and "Activity ideas" move to the suggestion area below with an "Or ask Rosie" divider. They're chat prompts, not logging actions. Logging is primary; chatting is secondary.

Floating Button — Chat + Log Entry Point

The home screen button needs to clearly communicate both capabilities: chatting with Rosie and quick logging. Currently the cycling text means parents might see "Ask RosieAI anything..." when they want to log, and vice versa.

F Split FAB

G Enhanced Pill
Single button, explicitly dual-purpose. "Chat or log with Rosie" replaces cycling text. Three colored dots (orange, purple, green) hint at log types. The 💜 heart ties it to Rosie's identity.
